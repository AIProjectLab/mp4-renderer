## Key Directories

### `/docs` — Documentation
All project documentation organized by topic:
- **IMPLEMENTATION_COMPLETE.md** — Complete feature guide
- **FEATURE_STATUS.md** — Current status of all features
- **INCIDENT_HISTORY.md** — Full incident report with learnings
- **AUDIO_DURATION_FIX.md** — Audio duration distribution fix
- **README.md** — Documentation index

**Use this folder to:**
- Understand what features are implemented
- Learn how to use each feature
- Troubleshoot issues
- Review incident history

### `/scenes` — Scene Animation Files
All HTML scene files for rendering:
- **stoicism_30scenes.html** — 30 stoic lessons
- **stoicism_10visuals.html** — 10 visual lessons
- **stoicism_cinematic15.html** — 15 cinematic scenes
- **finalgrok.html** — 21 scenes (65 minutes)
- **finalgrok2.html** — Alternative version
- **stoicsmGrok.html** — Groq integrated
- **grok2.html** — Groq integrated
- **README.md** — Scene files guide

**Use this folder to:**
- Upload scene files for rendering
- Create custom scene files
- Organize animation content

### `/public` — Static Assets
Static files served by the web server:
- **models/** — ML models (Whisper for transcription)
- **worker.js** — Web Worker for background transcription

### `/src` — Source Code
Application source files:
- **index.css** — Component styles
- **whisper-app.js** — Whisper integration

### `/subtitles` — Generated Subtitles
Auto-generated SRT subtitle files:
- Created automatically after Groq transcription
- Timestamped filenames prevent overwrites
- Standard SRT format (HH:MM:SS,mmm)

## How to Use

### For Users
1. Open `index.html` in browser
2. Upload scene file from `/scenes` folder
3. Configure and render
4. Check `/subtitles` for generated SRT files

### For Developers
1. Read `/docs/README.md` for documentation index
2. Check `/docs/IMPLEMENTATION_COMPLETE.md` for feature details
3. Review `/docs/INCIDENT_HISTORY.md` for learnings
4. Modify `app.js` for core logic changes
5. Add new scenes to `/scenes` folder

### For Troubleshooting
1. Check `/docs/INCIDENT_HISTORY.md` for known issues
2. Review `/docs/AUDIO_DURATION_FIX.md` for audio issues
3. Check browser console for errors
4. Verify scene file format in `/scenes/README.md`

---

## Adding New Content

### New Scene File
1. Create HTML file with SCENES and RENDERERS arrays
2. Place in `/scenes` folder
3. Update `/scenes/README.md` with description
4. Upload via app interface

### New Documentation
1. Create markdown file in `/docs` folder
2. Update `/docs/README.md` with reference
3. Link from relevant sections

### New Feature
1. Implement in `app.js`
2. Document in `/docs/IMPLEMENTATION_COMPLETE.md`
3. Add to `/docs/FEATURE_STATUS.md`
4. Update this file if structure changes

---

## Maintenance

### Regular Tasks
- Keep `/docs` updated with new features
- Archive old scene files if needed
- Clean up `/subtitles` folder periodically
- Update `/docs/README.md` with new documentation

### Backup
- Important files: `app.js`, `index.html`, `/docs`
- Scene files: `/scenes`
- Generated files: `/subtitles` (can be regenerated)
